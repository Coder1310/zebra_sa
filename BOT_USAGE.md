# Инструкция по Telegram-боту Zebra

Этот файл описывает полный сценарий работы с ботом:
- как подготовить окружение
- как запустить сервер и бота
- как создать игру
- как игрокам ходить
- как завершать игру

## 1. Что нужно перед запуском

Проект состоит из двух частей:
- FastAPI-сервер
- Telegram-бот на `aiogram`

Бот не работает сам по себе: ему нужен поднятый API-сервер.

Нужно подготовить:
- Python 3.10+ или совместимую версию
- Telegram bot token от BotFather
- установленный `venv` или другое окружение с зависимостями из `requirements.txt`

## 2. Установка зависимостей

Если окружение еще не готово:

```bash
python3 -m venv venv
. venv/bin/activate
pip install -r requirements.txt
```

## 3. Переменные окружения

Проект читает настройки из `.env`.

Минимальный пример:

```env
BOT_TOKEN=123456:replace_me
ZEBRA_API=http://127.0.0.1:8000
ZEBRA_HOST=127.0.0.1
ZEBRA_PORT=8000
```

Дополнительно можно настроить параметры партий:

```env
ZEBRA_PLAYERS=6
ZEBRA_HOUSES=6
ZEBRA_DAYS=50
ZEBRA_SHARE=meet
ZEBRA_NOISE=0.0
ZEBRA_GRAPH=ring
ZEBRA_LOBBY_DELAY_SEC=90
ZEBRA_TURN_DELAY_SEC=60
ZEBRA_VOTE_DELAY_SEC=45
```

Что означают основные параметры:
- `BOT_TOKEN`: токен Telegram-бота
- `ZEBRA_API`: адрес API-сервера, к которому ходит бот
- `ZEBRA_HOST`, `ZEBRA_PORT`: где поднимать FastAPI
- `ZEBRA_PLAYERS`: сколько игроков нужно для старта партии
- `ZEBRA_HOUSES`: сколько домов в мире
- `ZEBRA_DAYS`: максимальная длина партии в днях
- `ZEBRA_GRAPH`: граф перемещений, обычно `ring` или `full`
- `ZEBRA_LOBBY_DELAY_SEC`: сколько секунд держать лобби до автозапуска
- `ZEBRA_TURN_DELAY_SEC`: задержка между проверками ходов
- `ZEBRA_VOTE_DELAY_SEC`: длительность голосования за досрочное завершение

## 4. Как запустить

### Вариант A. Сервер и бот по отдельности

Сначала сервер:

```bash
. venv/bin/activate
./scripts/run_server.sh
```

Потом в отдельном терминале бот:

```bash
. venv/bin/activate
./scripts/run_bot.sh
```

### Вариант B. Быстрый совместный запуск

```bash
. venv/bin/activate
./run_all.sh
```

Этот способ поднимает сервер и затем запускает бота.

## 5. Как подготовить Telegram

Перед первой игрой:

1. Открой личный чат с ботом.
2. Нажми `/start`.
3. Добавь бота в групповой чат, где будет идти игра.
4. Разреши боту писать в группу.

Важно:
- каждый игрок должен хотя бы один раз нажать `/start` боту в личке
- без этого бот не сможет присылать личные ходы и приглашения

## 6. Основные команды и кнопки

В личке бот поддерживает:
- `/start`: зарегистрировать пользователя и показать меню
- `/help`: краткая справка

В группе можно использовать:
- `/game @user1 @user2 ...`: создать лобби и пригласить перечисленных пользователей
- `/end`: запросить завершение текущей игры

Также в меню есть кнопки:
- `🎮 Создать игру`
- `❓ Помощь`
- `🧹 Скрыть меню`

## 7. Как создать игру

Есть два способа.

### Способ 1. Через кнопку

1. В группе нажми `🎮 Создать игру`.
2. Бот попросит перечислить приглашенных пользователей.
3. Отправь список в формате:

```text
@user1 @user2 @user3
```

Если хочешь открыть лобби без списка приглашенных, отправь:

```text
-
```

После этого бот создаст сообщение-лобби с кнопками:
- `Join`
- `Leave`
- `Start now (host)`
- `Cancel (host)`

### Способ 2. Через команду

В группе отправь:

```text
/game @user1 @user2
```

Если usernames не указаны, лобби считается открытым.

## 8. Как работает лобби

После создания лобби:
- хост автоматически добавлен в список игроков
- приглашенные игроки могут нажать `Join`
- лишние игроки могут выйти через `Leave`
- хост может запустить игру сразу через `Start now (host)`
- если хост ничего не нажмет, игра стартует автоматически после `ZEBRA_LOBBY_DELAY_SEC`
- хост может отменить лобби через `Cancel (host)`

Если задан список приглашенных:
- войти могут только эти пользователи и сам хост
- приглашение в личку придет только тем, кто уже нажимал `/start`

## 9. Что происходит после старта игры

Когда лобби стартует:
- бот создает игровую сессию на FastAPI-сервере
- каждому человеку назначается роль
- в группу уходит сообщение о старте
- каждому игроку в личку приходит его приватное состояние и клавиатура действий

В личке игрок видит:
- свою роль
- текущий день
- свой дом
- текущее местоположение
- текущий `M1`
- сведения о поездке
- накопленные знания о домах

## 10. Как игрок делает ход

Игроки ходят в личных сообщениях с ботом.

Доступные действия зависят от состояния:

- `Stay`: остаться на месте
- `Left`: пойти в соседний дом слева
- `Right`: пойти в соседний дом справа
- `Choose destination`: выбрать любой дом, если используется граф `full`
- `Предложить обмен питомцами`: предложить обмен игроку в том же доме
- `Принять обмен`
- `Отказаться от обмена`
- `End game`: запустить голосование за завершение

Если игрок уже в пути:
- бот покажет, что остается ждать
- новый ход в этот момент не нужен

## 11. Как идет игровой цикл

Внутренняя логика такая:
- бот собирает ходы живых игроков
- когда все обязательные ходы сданы, он дергает `/game/{id}/step`
- сервер завершает игровой день
- бот рассылает отчеты по игрокам
- затем отправляет новый набор приватных ходов

В группе бот может сообщать:
- от кого сейчас ждутся действия
- что день завершен
- что игра завершилась

## 12. Обмен питомцами

Обмен работает только если:
- игроки находятся в одном доме
- оба являются human-игроками
- никто из них не находится в пути

Сценарий:

1. Игрок A нажимает `Предложить обмен питомцами`.
2. Бот показывает список доступных целей.
3. Игрок B получает кнопки `Принять` или `Отказаться`.
4. Результат учитывается на следующем шаге дня.

## 13. Как завершить игру

Есть два сценария.

### Обычное завершение

Игра завершится сама:
- когда дойдет до лимита дней
- когда сервер решит завершить партию по логике симуляции

### Досрочное завершение

Игрок может:
- нажать `End game` в личке
- или отправить `/end` в группе

Если живых игроков больше одного:
- бот запускает голосование
- каждому игроку приходит кнопка `Завершить` или `Продолжать`
- нужно простое большинство голосов `Да`

Если игрок один:
- игра завершается сразу без голосования

## 14. Что приходит после завершения

После окончания игры бот:
- отправляет итог в группу
- пытается отправить итог каждому игроку в личку
- собирает архив логов игры

Обычно в архив попадают:
- `metrics`
- `metrics_ext`
- `csv`
- `xml`

Архив сохраняется и отправляется как `game_<game_id>.zip`.

## 15. Типовой сценарий использования

Коротко полный поток выглядит так:

1. Подними сервер.
2. Подними бота.
3. Все игроки нажимают `/start` в личке с ботом.
4. Добавь бота в групповой чат.
5. В группе создай лобби через `🎮 Создать игру` или `/game`.
6. Игроки нажимают `Join`.
7. Хост жмет `Start now`, либо ждет автозапуска.
8. Игроки делают ходы в личке.
9. Бот шаг за шагом ведет партию.
10. После завершения бот рассылает итог и архив.

## 16. Если что-то не работает

Проверь по порядку:

1. Запущен ли FastAPI-сервер.
2. Совпадает ли `ZEBRA_API` у бота с реальным адресом сервера.
3. Заполнен ли `BOT_TOKEN`.
4. Нажали ли игроки `/start` в личке.
5. Есть ли у приглашенных корректные `@username`.
6. Не занят ли уже этот групповой чат другой игрой или лобби.

Типичные симптомы:
- `BOT_TOKEN is not set`: не заполнен токен
- бот не пишет в личку: пользователь не нажал `/start`
- игра не стартует: сервер не поднят или `ZEBRA_API` указывает не туда
- нет приглашения по username: пользователь не запускал бота раньше или username отсутствует
