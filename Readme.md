# ZEBRA

Прототип тактической игры и симулятора ситуационной осведомленности на сюжете Zebra Puzzle.

Проект собран как клиент-серверное приложение. Сервер держит игровую сессию, считает состояние мира и формирует логи. Telegram-бот отвечает за лобби, приглашения, приватные ходы и выдачу игроку его текущей картины мира. Отдельный слой анализа работает с логами и метрикой `M1(t)`.

## Что уже есть

- FastAPI сервер с REST API для создания игры, получения состояния, приема действий, шага по дням и завершения партии
- Telegram Bot для группового лобби и личных ходов
- batch-симуляция без Telegram
- логирование в CSV и XML
- файл метрик `M1(t)` по дням для каждого участника
- базовые скрипты анализа логов и построения графиков

## Идея модели

На острове есть 6 домов. У каждого дома есть цвет. В домах живут шесть островитян разных национальностей. У каждого есть напиток, питомец и марка сигарет.

Игрок не видит весь мир сразу. Он узнает факты через поездки, встречи и обмен информацией. По итогам каждого дня считается `M1` - доля корректно известных фактов о том, кто где живет, что пьет, что курит и какого питомца держит.
