# Zebra Puzzle - тактическая игра и симулятор для исследования situational awareness

`Zebra Puzzle` - это прототип исследовательского игрового сервиса, в котором классическая логическая задача про островитян превращена в динамическую тактическую среду. Игроки перемещаются между домами, наблюдают события, встречаются друг с другом, обмениваются знаниями и в некоторых сценариях могут менять питомцев или дома. Главная цель проекта - не просто провести игру, а измерить, как по ходу партии меняется situational awareness (SA) участников.

Проект собран как клиент-серверная система:
- **FastAPI** отвечает за API и состояние игровых сессий
- **симуляционное ядро** реализует правила мира и пошаговую дискретно-событийную модель
- **Telegram Bot** используется как интерфейс игрока: лобби, приглашения, приватные ходы, уведомления и завершение игры
- **analysis/** содержит утилиты для обработки логов и построения графиков `SA(t)`

Такой формат позволяет использовать один и тот же мир и одни и те же правила в двух режимах:
- как **интерактивную игру** с живыми участниками
- как **экспериментальный стенд** для пакетных прогонов и сравнения стратегий

## Что именно моделируется

Игровой мир состоит из домов и жителей с набором атрибутов:
- цвет дома
- национальность
- напиток
- питомец
- сигареты

В отличие от классической статической Zebra Puzzle, здесь мир меняется во времени:
- игроки перемещаются между домами
- поездка занимает несколько дней
- при встречах могут открываться новые факты
- при включенном режиме обмена знаниями часть информации распространяется между участниками
- при некоторых настройках возможны обмены питомцами и домами
- можно включать шум наблюдений
- граф перемещений может быть `ring` или `full`

За счет этого задача превращается в наблюдение за динамической системой, где важно не просто "знать ответ", а понимать, **как именно участник пришел к этому знанию** и сколько информации он накопил к каждому дню.

## Исследовательская цель

Проект делался как основа для исследования **situational awareness** в игровой среде.

В текущей версии используются две ключевые метрики:

### M1
Доля корректно известных фактов о мире для конкретного участника.

Идея простая: у каждого игрока есть частичная картина мира. Чем больше верных фактов он накопил, тем выше его `M1`.

### M2
Частная метрика про зебру: известно ли игроку, **кто является владельцем зебры**.

Эта метрика нужна как более жесткий и понятный индикатор. Игрок может знать много локальных фактов, но все еще не понимать главный интересующий объект. Поэтому `M1` и `M2` полезно анализировать вместе.

## Ключевые возможности репозитория

### 1. REST API для игровых сессий
Сервер умеет:
- создавать сессии
- запускать batch-симуляции
- создавать интерактивные игры
- принимать действия игроков
- выполнять шаг симуляции по дням
- отдавать состояние игры и состояние конкретного игрока
- завершать игру и возвращать файлы логирования

### 2. Telegram-бот
Бот используется как внешний интерфейс и поддерживает:
- создание лобби в групповом чате
- вход и выход игроков из лобби
- старт игры
- приватные ходы через личные сообщения
- отображение текущего состояния игрока
- голосование за завершение партии
- отправку итогов и архивов логов после завершения игры

### 3. Batch-симуляция
Игру можно запускать без Telegram и без ручных действий игроков. Это удобно для:
- серийных экспериментов
- сравнения настроек среды
- отладки метрик
- проверки влияния стратегий и параметров

### 4. Логирование и анализ
По завершении симуляции формируются:
- событийный лог в CSV
- событийный лог в XML
- таблица метрик по дням
- расширенная таблица метрик с `M1`, `M2` и дополнительными признаками

На основе этих файлов можно строить графики `SA(t)` и сравнивать разные сценарии.
